# 08 · 服务端编排最小示例

- 对应文档：`BE/08-服务端编排与任务流.md`
- 演示：DAG 调度、无依赖节点并行、单步超时、失败重试退避、全局并发限制、耗时统计。
- 只用 Python 标准库 asyncio，无需安装依赖。

## 运行

```bash
python orchestrator.py
```

## 看什么

- `bg` 节点有 50% 概率失败，会触发重试退避，观察它最终成功或抛错。
- `matte` 和 `bg` 在同一批并行执行（总耗时取决于更慢的那个，而不是相加）。
- `compose` 等两者都完成才开始，`upscale` 最后跑——这就是按 DAG 依赖调度。
- 每个节点打印自己的耗时，最后打印总耗时。

## 可以动手改的点

- 把 `SEM = asyncio.Semaphore(3)` 改成 1，观察节点变成几乎串行。
- 调大某个 `fake_call` 的 `base_ms` 超过节点 `timeout`，观察超时后重试直至失败。
- 加一个新节点并设置依赖，观察调度顺序变化。
