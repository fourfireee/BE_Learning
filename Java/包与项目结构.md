# Java 包与项目结构

- 这一篇补 Java 项目最容易让 C++/native 背景困惑的一块：类放在哪、包名是什么、`import` 做了什么、运行时怎么找到类。
- 在后端项目里，包结构就是代码的“命名空间 + 模块边界”，非常重要。

## package：命名空间

- `package` 写在 Java 文件第一行，表示这个类属于哪个包。
- 包名通常全小写，用反向域名开头，如 `com.example.user`。
- 类的完整名字 = 包名 + 类名，例如 `com.example.user.UserService`。
- 类比 C++：`package` 很像 `namespace`，但 Java 还强约定“包名和目录结构对应”。

```java
package com.example.model;

public class User {
    private final String name;
    private final int age;

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

- 上面这个类通常放在：

```txt
src/main/java/com/example/model/User.java
```

## import：简化类名

- `import` 只是让你少写完整类名，不等于 C++ 的 `#include`。
- Java 没有头文件，也不会把另一个文件内容拷贝进来。❓

```java
package com.example.app;

import com.example.model.User;

public class Main {
    public static void main(String[] args) {
        User u = new User("Alice", 18);
        System.out.println(u.name());
    }
}
```

- 如果不写 import，也可以写完整类名：

```java
com.example.model.User u = new com.example.model.User("Alice", 18);
```

## 文件名和 public 类

- 一个 `.java` 文件里最多只能有一个 `public` 顶层类。
- `public` 类名必须和文件名一致：`public class User` 必须放在 `User.java`。
- 非 public 的辅助类可以放在同一个文件里，但真实后端项目通常一个主要类一个文件，更清楚。

## classpath：运行时找类的路径

- 编译后的 `.class` 文件也会按包名放进目录。
- `classpath` 告诉 JVM 从哪些根目录或 jar 包里找类。
- 类比 C++：classpath 有点像“动态链接搜索路径 + include/link 配置”的合体，但它找的是 `.class` / jar。

```bash
# -d out 表示把编译产物输出到 out 目录，并按 package 创建子目录
javac -d out src/com/example/model/User.java src/com/example/app/Main.java

# -cp out 表示从 out 这个 classpath root 找类
java -cp out com.example.app.Main
```

- 注意运行时写的是完整类名 `com.example.app.Main`，不是文件路径，也不是 `Main.class`。

## Maven 标准目录

- 真实项目里不直接手写 classpath，Maven / Gradle 会帮你管理。
- Maven 约定：

```txt
my-app/
  pom.xml
  src/
    main/
      java/          # Java 源码
      resources/     # 配置文件、模板等资源
    test/
      java/          # 测试源码
      resources/     # 测试资源
```

- `src/main/java` 是源码根目录，不是包名的一部分。
- `com/example/app/Main.java` 才对应 `package com.example.app;`。

## package-private：包内可见

- Java 不写访问修饰符时，不是 public，而是 package-private：同一个包内可见，包外不可见。
- 这很适合隐藏模块内部实现，只把真正的 API 设成 `public`。

```java
class InternalParser {
    // 只给同包代码使用，包外看不到这个类
}
```

## 后端项目常见分层

- 简单后端项目常按职责拆包：
- `controller`：HTTP 入口，接收请求、返回响应。
- `service`：业务逻辑。
- `repository` / `dao`：数据访问。
- `model` / `entity`：领域对象或数据库实体。
- `dto`：接口输入输出的数据对象。
- `config`：配置类。

```txt
com.example.demo/
  controller/
  service/
  repository/
  model/
  dto/
  config/
```

- 不要为了分层而机械搬运代码。包结构的目标是让“入口、业务、数据访问”边界清楚。
