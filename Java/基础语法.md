# Java 基础语法

- 这一篇过一遍 Java 的基本语法，重点放在“和 C++ 不一样的地方”，你已经会的就不展开了。

## 基本类型

- Java 的基本类型大小是固定的，不像 C++ 依赖平台。这点对你很友好，没有 `int 到底几位` 的困扰。

- `boolean`：true / false，不能和整数互换（C++ 里 0/1 可当 bool，Java 不行）。
- `byte`：8 位有符号整数。
- `short`：16 位。
- `int`：32 位，默认整数类型。
- `long`：64 位，字面量加后缀 `L`，如 `100L`。
- `float`：32 位浮点，字面量加后缀 `f`，如 `1.0f`。
- `double`：64 位浮点，默认浮点类型。
- `char`：16 位，存的是 UTF-16 code unit，不是 C++ 那种 1 字节。注意一个 emoji 或部分生僻字可能需要两个 `char` 才能表示。
    - UTF = Unicode Transformation Format，Unicode 的编码方式，把字符（码点）转成实际存储的字节。常见有 UTF-8（变长 1~4 字节，网络/文件主流）和 UTF-16（变长 2 或 4 字节，Java 字符串内部用的就是它）。

## 基本类型 vs 引用类型

- Java 把数据分两类，这是理解 Java 的关键分水岭：
- 基本类型（上面那些）：变量直接存值，存在栈上，赋值是拷贝值。
- 引用类型（对象、数组、String 等）：变量存的是“指向堆上对象的引用”，赋值拷贝的是引用，不是对象本身。
- 类比 C++：引用类型变量更像“永远是指针/智能指针”，但没有 `*`、`->` 这些符号，统一用 `.` 访问。

```java
int a = 10;
int b = a;      // 拷贝值，b 改了不影响 a
b = 20;         // a 仍是 10

int[] arr1 = {1, 2, 3};
int[] arr2 = arr1;   // 拷贝引用，两个变量指向同一个数组
arr2[0] = 99;        // arr1[0] 也变成 99，因为是同一个对象
```

## 包装类型与自动装箱

- 每个基本类型都有一个对应的“包装类”（对象版本）：`int → Integer`、`double → Double`、`boolean → Boolean` 等。
- 为什么需要：很多 Java 容器（如 `List`）只能装对象，不能装基本类型，所以需要包装版本。
- `自动装箱 (autoboxing)`：编译器自动在 `int` 和 `Integer` 之间转换，写起来无感。

```java
// 自动装箱：int 自动变成 Integer 对象
Integer boxed = 42;
// 自动拆箱：Integer 自动变回 int
int raw = boxed;
```

- 一个坑：包装类型是对象，比较值要用 `.equals()`，用 `==` 比的是引用是否同一个对象。`Integer` 对小整数有缓存，`==` 有时看似能用，但不要依赖这个细节。

## null 与空指针

- `null` 表示“这个引用不指向任何对象”，只能赋给引用类型，不能赋给 `int` 这种基本类型。
- 对 `null` 调方法或访问字段会抛 `NullPointerException`，这是 Java 后端里非常常见的错误。
- 类比 C++：`null` 像空指针；区别是 Java 没有野指针和悬垂指针，但空引用仍然会出错。
    - 野指针（wild pointer）：从没正确初始化过的指针，里面是随机垃圾地址，指向哪儿不知道。比如 `int* p;` 没赋值就 `*p`。
    - 悬垂指针（dangling pointer）：曾经指向合法内存，但那块内存已被释放（`free`/`delete`）或离开作用域，指针却还指着那个已失效的地址。
    - 一句话区别：野指针是「一开始就没指对」，悬垂指针是「本来指对了、但目标后来没了」。两者解引用都是未定义行为。
    - 为什么 Java 没有：Java 不暴露裸指针、引用必须初始化，且内存由 GC（垃圾回收）统一管理、不手动 `free`，所以这两类问题天然不存在；但引用可以是 `null`，访问它仍会抛 `NullPointerException`。

```java
String name = null;
// name.length(); // 会抛 NullPointerException
```

## String 是特殊的引用类型

- `String` 是对象，不是基本类型。它“不可变”：一旦创建，内容不能改，任何“修改”都会生成新对象。
- 比较内容必须用 `.equals()`，不要用 `==`：

```java
String s1 = "hello";
String s2 = new String("hello");
boolean r1 = (s1 == s2);        // false：两个不同对象（比引用）
boolean r2 = s1.equals(s2);     // true：内容相同（比值）
```

- 频繁拼接字符串用 `StringBuilder`，避免每次拼接都生成新 String（类似 C++ 里避免反复构造临时 string）。

```java
final int N = 100000;

// 反例：循环里用 += 拼接。String 不可变，每次 += 都会新建一个 String 对象，
// 第 i 次要复制前面已有的全部字符，N 次累计是 O(n^2)，还制造大量垃圾。
long t1 = System.nanoTime();                 // 记录开始时间（纳秒）
String s = "";
for (int i = 0; i < N; i++) {
    s += i;                    // 每次都生成一个新 String
}
long cost1 = System.nanoTime() - t1;         // 耗时 = 结束 - 开始

// 正解：StringBuilder 内部维护一个可变的字符缓冲区，append 只是往后追加，
// 不重复复制已有内容，整体 O(n)；最后 toString() 一次性得到结果。
long t2 = System.nanoTime();
StringBuilder sb = new StringBuilder();
for (int i = 0; i < N; i++) {
    sb.append(i);              // 原地追加，不新建对象
}
String result = sb.toString();
long cost2 = System.nanoTime() - t2;

// 1_000_000 是把纳秒换算成毫秒（ms）。下划线只是数字分隔符，方便阅读，不影响数值。
System.out.println("+= 拼接耗时：" + cost1 / 1_000_000 + " ms");
System.out.println("StringBuilder 耗时：" + cost2 / 1_000_000 + " ms");
// 典型结果：+= 要几百甚至上千 ms，StringBuilder 通常只要几 ms，差距随 N 增大而急剧拉大。
```

    - 注意：只是少量几段固定拼接（如 `"a" + b + "c"`）不用特意上 StringBuilder——编译器会自动优化，可读性优先。瓶颈只在「循环 / 大量拼接」时才明显。

## 变量与控制流

- 控制流语法和 C/C++ 几乎一样：`if/else`、`for`、`while`、`switch`，这里只说差异点。

- 增强 for（for-each），遍历容器/数组很常用：

```java
int[] nums = {1, 2, 3};
// 类似 C++ 的 range-based for: for (int n : nums)
for (int n : nums) {
    System.out.println(n);
}
```

- switch 支持字符串，且 Java 14+ 有更安全的箭头写法（不会“掉落”到下一个 case）：

```java
String cmd = "add";
switch (cmd) {
    case "add" -> System.out.println("新增");
    case "del" -> System.out.println("删除");
    default -> System.out.println("未知命令");
}
```

- `var` 局部类型推断（Java 10+），类似 C++ 的 `auto`，只能用于局部变量：

```java
var list = new java.util.ArrayList<String>(); // 编译器推断出类型
```

- `ArrayList` 和 `[]` 数组的区别（都「按下标存一组元素」，但定位不同）：
    - 长度：数组 `String[] a = new String[3]` 创建后长度固定、不能增删；`ArrayList` 可动态 `add` / `remove`，底层自动扩容（类似 C++ 的 `std::vector`，数组则像定长 C 数组）。
    - 元素类型：数组能直接装基本类型（`int[]`，无装箱开销）；`ArrayList<Integer>` 只能装对象，基本类型会被自动装箱成包装类（有少量开销）。
    - 用法：数组用 `a[i]` 取值、`a.length` 看长度；`ArrayList` 用 `list.get(i)` / `list.size()`，还自带 `contains`、`indexOf`、遍历等一堆方法。
    - 怎么选：长度固定、追求极致性能（尤其大量基本类型）用数组；长度会变、要频繁增删或用现成方法，用 `ArrayList`（日常更常用）。

## 数组

- 数组是引用类型，长度固定，创建时确定。访问越界会抛 `ArrayIndexOutOfBoundsException`，不会像 C++ 那样静默踩内存。

```java
int[] a = new int[3];      // 默认值全为 0
int[] b = {10, 20, 30};    // 直接初始化
int len = b.length;        // 注意是属性 length，不是 size()/函数
```

## final：不可重新赋值

- `final` 修饰变量，表示这个变量只能赋值一次，之后不能再指向别的值/对象。
- 对引用类型来说，`final` 限制的是“引用不能换”，不是“对象内容不能改”。这点和 C++ 的 `T* const` 更像，不等于 `const T*`。

```java
final int x = 10;
// x = 20; // 编译错误

final int[] nums = {1, 2, 3};
nums[0] = 99;        // 可以，数组对象本身仍然可变
// nums = new int[3]; // 编译错误，引用不能改指向
```

## 和 C++ 的语法差异速查

- 没有头文件、没有 `#include`，用 `import` 引入其它包的类。
- 没有指针运算、没有 `&` 取地址、没有手动 `new/delete` 配对（只有 `new`，回收交给 GC）。
    - GC（Garbage Collection，垃圾回收）简述：JVM 自动帮你回收「不再被使用的对象」内存，你只管 `new`，不用也不能手动 `delete`。
    - 怎么判断「不再使用」：从一组根（如栈上的局部变量、静态字段）出发能顺着引用链到达的对象算「存活」，到达不了的就是垃圾，可被回收。（注意：是「可达性」，不是 C++ 智能指针那种引用计数，所以循环引用也能被正确回收。）
    - 大致怎么做：主流是「分代回收」——新对象大多很快就死，放在新生代频繁快速回收；活得久的晋升到老年代，较少回收。这样省力高效。
    - 代价：回收时可能有短暂停顿（STW，Stop-The-World，暂停业务线程）；好处是基本告别内存泄漏和悬垂指针（但仍可能因「一直被引用却不再用」造成逻辑上的内存泄漏）。
    - 这里先有个印象即可，细节（垃圾回收器种类、调优）后面再展开。
- 没有多继承（类只能单继承），但可以实现多个接口。
    - C++ 里一个类可以同时继承多个父类（多继承），但这会带来「菱形继承」等歧义问题（两条路径继承到同一个基类，到底用哪份）。Java 干脆禁掉：`extends` 只能跟一个父类。
    - 但 `implements` 可以跟多个接口，用逗号隔开：`class Foo extends Base implements A, B, C`。意思是「Foo 是一个 Base，同时承诺具备 A、B、C 这几套能力」。
    - 为什么多接口不会有多继承的歧义：接口传统上只规定「有哪些方法」（一份契约），不带实现和字段，所以实现多个接口只是「答应多套契约」，不存在「继承到两份冲突的实现」。
    - 类比：单继承是「我属于哪一类（is-a）」只能选一个；实现接口是「我会做哪些事（can-do）」可以会很多种。比如 `class Dog extends Animal implements Swimmable, Runnable`：它本质是动物，但同时会游泳、会跑。
    - 补充：Java 8 起接口可以带「默认方法（default）」自带实现，于是也可能出现两个接口给了同名默认方法的冲突——这时编译器会强制你在类里手动覆写、指明用哪个，仍然不留歧义。
- 没有全局函数：所有函数都必须写在某个类里（叫方法）。
- 一切非基本类型都是“引用”，统一用 `.` 访问成员，没有 `->`。
