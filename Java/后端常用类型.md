# Java 后端常用类型

- 这一篇不是新语法，而是后端项目里经常出现的标准库类型。
- 这些类型用对了，可以少写很多脆弱的字符串、浮点数和 null 判断。

## Optional

- `Optional<T>` 表示“可能有值，也可能没有值”。
- 常用于方法返回值，提醒调用方处理不存在的情况。
- 不建议滥用在字段、DTO 入参、集合元素里；业务对象里直接用清晰的 null 策略通常更简单。

```java
import java.util.Optional;

Optional<String> findNameById(long id) {
    if (id == 1) return Optional.of("Alice");
    return Optional.empty();
}

String name = findNameById(2).orElse("unknown");
```

- `Optional.of(value)` 不接受 null；可能为 null 时用 `Optional.ofNullable(value)`。

## BigDecimal

- 金额、价格、精确小数不要用 `double` / `float`，用 `BigDecimal`。
- 浮点数是二进制近似，适合图形、物理、统计，不适合钱。

```java
import java.math.BigDecimal;
import java.math.RoundingMode;

BigDecimal price = new BigDecimal("19.90");
BigDecimal count = new BigDecimal("3");
BigDecimal total = price.multiply(count);
BigDecimal avg = total.divide(new BigDecimal("2"), 2, RoundingMode.HALF_UP);
```

- 优先用字符串构造 `new BigDecimal("0.1")`，不要用 `new BigDecimal(0.1)`。

## java.time

- 新代码使用 `java.time` 包，少用老的 `Date` / `Calendar`。
- `Instant`：时间线上的一个绝对时刻，适合存数据库、事件时间。
- `LocalDate`：只有日期，没有时间和时区，如生日、账单日期。
- `LocalDateTime`：日期 + 时间，但没有时区。
- `ZonedDateTime`：日期 + 时间 + 时区。
- `Duration`：两个时间点之间的时长。

```java
import java.time.Instant;
import java.time.LocalDate;
import java.time.ZoneId;
import java.time.ZonedDateTime;

Instant now = Instant.now();
LocalDate today = LocalDate.now();
ZonedDateTime shanghai = now.atZone(ZoneId.of("Asia/Shanghai"));
```

- 后端接口和数据库里尽量明确时区策略。常见做法是内部存 `Instant` 或 UTC 时间，展示时再转用户时区。

## enum

- 固定有限选项用 `enum`，不要散落字符串常量。
- 好处是编译器能检查拼写，switch 也更安全。

```java
public enum OrderStatus {
    CREATED,
    PAID,
    CANCELLED
}
```

```java
OrderStatus status = OrderStatus.PAID;
switch (status) {
    case CREATED -> System.out.println("待支付");
    case PAID -> System.out.println("已支付");
    case CANCELLED -> System.out.println("已取消");
}
```

## UUID

- `UUID` 常用于生成不容易冲突的请求 ID、业务 ID、追踪 ID。
- 它不是递增 ID，不适合所有数据库主键场景，但很适合跨系统生成唯一标识。

```java
import java.util.UUID;

String requestId = UUID.randomUUID().toString();
```

## record 和 DTO

- `record` 适合做不可变数据载体，比如接口返回对象、事件消息、Map key。
- 它默认按字段生成构造器、访问方法、equals / hashCode / toString。

```java
public record UserResponse(long id, String name, String email) {}
```

- 注意 record 是浅不可变：字段引用不能换，但字段对象本身如果可变，内部内容仍可能变。
