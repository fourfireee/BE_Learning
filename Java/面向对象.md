# Java 面向对象

- Java 是彻底的面向对象语言：所有代码都写在类里，没有游离的全局函数和全局变量。
- 你有 C++ 的 OOP 基础，这一篇重点讲 Java 的差异和约定。

## 类与对象

- `class` 定义一类东西的结构和行为；`new` 创建出来的具体实例就是对象（在堆上）。
- 类的成员：字段（数据）、方法（行为）、构造器（创建时初始化）。

```java
public class User {
    // 字段（成员变量），private 表示只能类内部访问
    private String name;
    private int age;

    // 构造器：名字和类名相同，没有返回类型。new User(...) 时调用
    public User(String name, int age) {
        // this 指向当前对象，用来区分同名的参数和字段
        this.name = name;
        this.age = age;
    }

    // 方法：对外提供的行为
    public String describe() {
        return name + " (" + age + ")";
    }
}
```

```java
// 创建对象一定用 new，返回的是堆上对象的引用
User u = new User("Alice", 18);
System.out.println(u.describe());
// 不需要手动 delete，GC 会在没人引用后自动回收
```

## 封装与访问修饰符

- 习惯做法：字段设为 `private`，通过方法（getter/setter）暴露，外部不能直接乱改内部状态。
- 四种访问级别：
- `public`：所有地方可访问。
- `protected`：同包 + 子类可访问。
- 默认（不写修饰符）：同包可访问（叫 package-private）。
- `private`：仅本类可访问。

## 继承

- 用 `extends` 继承，只能单继承（一个类只有一个父类），这是和 C++ 的重要区别（C++ 支持多继承）。
- `super` 调用父类构造器或父类方法。
- `@Override` 注解标记“这是在重写父类方法”，写错签名编译器会报错，建议总是加上。

```java
public class Animal {
    protected String name;
    public Animal(String name) { this.name = name; }
    public String speak() { return "..."; }
}

public class Dog extends Animal {
    public Dog(String name) {
        super(name); // 必须先调用父类构造器
    }

    @Override
    public String speak() { // 重写父类方法
        return name + ": Woof";
    }
}
```

## 多态

- 同一个父类引用，指向不同子类对象，调用同一个方法表现不同。这是 OOP 的核心，和 C++ 虚函数一个意思。
- 区别：C++ 要显式写 `virtual` 才有动态绑定；Java 的实例方法默认就是虚的（默认动态绑定），不需要关键字。

```java
Animal a = new Dog("Rex"); // 父类引用指向子类对象
System.out.println(a.speak()); // 实际调用 Dog 的版本，输出 "Rex: Woof"
```

## 抽象类

- `abstract class`：不能直接 new，用来作为基类定义“部分实现 + 部分留给子类实现”的模板。
- `abstract` 方法只有声明没有实现，子类必须实现，类似 C++ 的纯虚函数。

```java
public abstract class Shape {
    public abstract double area(); // 抽象方法，子类必须实现

    public String report() { // 普通方法，子类直接复用
        return "面积是 " + area();
    }
}
```

## 接口

- `interface`：定义“能力契约”，只规定有哪些方法，不关心怎么实现。类用 `implements` 实现接口。
- 一个类可以实现多个接口，这就是 Java 替代“多继承”的方式。
- 接口里的方法默认是抽象的；Java 8 起也允许写 `default` 方法（带默认实现）。

```java
public interface Drawable {
    void draw(); // 抽象方法
}

public interface Clickable {
    void onClick();
}

// 一个类可以同时实现多个接口
public class Button implements Drawable, Clickable {
    @Override public void draw() { System.out.println("画按钮"); }
    @Override public void onClick() { System.out.println("点击了"); }
}
```

## 抽象类 vs 接口怎么选

- 抽象类：表达“is-a”关系，且需要共享一些字段或已实现的方法。一个类只能继承一个抽象类。
- 接口：表达“能做什么”的能力，可多实现，更灵活。优先用接口做解耦。
- 经验法则：先用接口定义契约，需要复用实现细节时再考虑抽象类。

## static 成员

- `static` 修饰的字段/方法属于“类本身”，而不是某个对象，所有实例共享，用类名直接访问。
- 类似 C++ 的静态成员。`main` 方法就是 static 的，所以不用先 new 对象就能调用。

```java
public class Counter {
    private static int total = 0; // 所有对象共享这一份
    public Counter() { total++; }
    public static int getTotal() { return total; } // 用 Counter.getTotal() 调用
}
```

## record：简洁的数据类

- Java 16+ 正式支持的 `record` 用来快速定义“只装数据”的不可变类，自动生成构造器、getter、equals / hashCode / toString 等，省去样板代码。
- 很适合做后端的 DTO / 值对象。

```java
// 一行定义一个不可变数据类，自动有构造器和 name()/age() 访问方法
public record UserDTO(String name, int age) {}
```

## equals、hashCode、toString

- `Object` 是所有类的根类，所有对象都有 `equals()`、`hashCode()`、`toString()`。
- 默认 `equals()` 比的是引用是否同一个对象；如果你希望按字段内容比较，就要重写。
- 自定义对象放进 `HashSet` 或作为 `HashMap` 的 key 时，必须让 `equals()` 和 `hashCode()` 语义一致。
- `record` 会自动生成这些方法，所以很适合做后端 DTO、值对象、Map key。

## 没有析构函数

- Java 没有 C++ 那种确定时机执行的析构函数，不要指望对象被 GC 时自动释放文件、连接、Socket。
- 资源释放交给 `try-with-resources`，具体放在[异常处理](异常处理.md)和[IO与文件](IO与文件.md)里看。
