# Java 并发

- 后端服务天生要并发处理大量请求，所以并发是 Java 后端的高频核心。
- 你有 native 多线程经验，这篇重点讲 Java 的并发模型和工具。

## 线程基础

- `Thread` 代表一个线程；最常见的是给它一个 `Runnable`（一段要执行的任务）。

```java
// 方式一：传入 Runnable（一个无参无返回的任务）
Thread t = new Thread(() -> {
    System.out.println("在新线程里运行");
});
t.start();   // start() 才会真正开新线程；直接调 run() 只是普通方法调用
t.join();    // 等待这个线程结束（类似 std::thread::join）
```

- 注意区别：`start()` 开新线程执行；`run()` 只是在当前线程同步调用，不会并发。

## 共享状态与竞态

- 多线程同时读写同一份数据，会出现“竞态条件”，结果不可预测。这点和 C++ 一样。

```java
class Counter {
    private int count = 0;
    public void inc() { count++; } // count++ 不是原子操作，多线程下会丢更新
}
```

- `count++` 实际是“读-加-写”三步，多个线程交错执行会互相覆盖。

## synchronized：内置锁

- 给方法或代码块加 `synchronized`，同一时刻只有一个线程能进入，保证互斥。类似 C++ 的 `mutex` + `lock_guard`，但语法内置。

```java
class SafeCounter {
    private int count = 0;
    // 加锁后，同一对象上的 inc() 同一时间只能有一个线程执行
    public synchronized void inc() { count++; }

    public void incBlock() {
        synchronized (this) {  // 也可以只锁一段代码块
            count++;
        }
    }
}
```

## volatile：可见性

- `volatile` 保证一个变量的修改对其它线程立即可见（禁止线程缓存旧值），但它不保证原子性。
- 适合“一个写、多个读”的状态标志，比如停止标志位。

```java
private volatile boolean running = true; // 其它线程改成 false 后，本线程能立刻看到
```

## Java 内存模型：happens-before

- 多线程不只是“谁先执行”的问题，还有“一个线程写入的值，另一个线程什么时候能看见”的问题。
- Java 用 Java Memory Model 规定可见性和重排序规则，核心直觉叫 `happens-before`：如果 A happens-before B，那么 B 一定能看见 A 之前的写入。
- 常见能建立 happens-before 的动作：
- 释放锁 happens-before 后续获取同一把锁。
- 写 `volatile` 变量 happens-before 后续读同一个 `volatile` 变量。
- `Thread.start()` 之前的写入，对新线程可见。
- 一个线程结束 happens-before 另一个线程从 `join()` 返回。
- 初学不用背规则，先记住：共享可变状态要通过锁、volatile、原子类、线程安全容器或 Future/CompletableFuture 这类同步工具传递。

## 原子类

- `java.util.concurrent.atomic` 提供无锁的原子操作，性能比 synchronized 好，适合简单计数。

```java
import java.util.concurrent.atomic.AtomicInteger;

AtomicInteger counter = new AtomicInteger(0);
counter.incrementAndGet(); // 原子自增，多线程安全，无需加锁
```

## 线程池：别手动 new 线程

- 实际后端几乎不直接 `new Thread`，而是用线程池复用线程，避免频繁创建销毁的开销，并控制并发数。

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

ExecutorService pool = Executors.newFixedThreadPool(4); // 固定 4 个线程，适合教学示例
pool.submit(() -> System.out.println("任务1"));          // 提交任务
pool.submit(() -> System.out.println("任务2"));
pool.shutdown(); // 不再接收新任务，等已提交的执行完
```

- 生产代码里要注意队列大小、拒绝策略、线程命名和异常处理；很多项目会直接配置 `ThreadPoolExecutor`，或交给 Spring 管理。

## 异步结果：Future / CompletableFuture

- 提交任务后想拿返回值，用 `Future`；要做链式异步编排，用 `CompletableFuture`。

```java
import java.util.concurrent.CompletableFuture;

// 异步执行并在完成后处理结果，类似 JS 的 Promise.then
CompletableFuture
    .supplyAsync(() -> 1 + 1)            // 异步计算
    .thenApply(sum -> sum * 10)          // 拿到结果继续处理
    .thenAccept(r -> System.out.println("结果: " + r)); // 消费最终结果
```

## 并发容器

- 普通 `HashMap` 多线程下不安全，要用并发版本：
- `ConcurrentHashMap`：线程安全的高性能 Map，后端缓存常用。
- `CopyOnWriteArrayList`：读多写少场景的线程安全 List。
- `Hashtable` / `Collections.synchronizedMap` 比较老，通常不作为新代码首选。

```java
import java.util.concurrent.ConcurrentHashMap;
ConcurrentHashMap<String, Integer> cache = new ConcurrentHashMap<>();
cache.put("a", 1); // 多线程并发读写安全
```

## 虚拟线程

- Java 21 正式引入虚拟线程（virtual thread），可以把大量阻塞式 IO 任务映射到少量平台线程上，适合后端“等数据库/等网络”的高并发场景。
- 它解决的是“线程数量和阻塞成本”的问题，不会自动解决共享状态、锁竞争、竞态条件。

```java
try (var executor = java.util.concurrent.Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> System.out.println(Thread.currentThread()));
}
```

## 经验法则

- 优先“不共享”：能用局部变量、无状态对象就别共享可变状态，从根上避免并发问题。
- 必须共享时：简单计数用原子类；复杂逻辑用锁，但锁的范围尽量小。
- 用线程池而非裸线程；用并发容器而非自己加锁包装普通容器。
- 现代框架（如 Spring）大多帮你管好线程，业务代码尽量写成无状态、线程安全的。
