# 背景

- 我近期要学习和使用后端开发。现在已有的后端语言，主要是Java。后端知识我另外学，这里我主要是要学习Java。
- 我之前的开发背景：计算机图形学、native 渲染引擎及工具、以及运行在 native 引擎之上的 shader 特效。

# 写作和示例要求

- 用尽量通俗易懂的语言来精简地描述概念和计算原理。
- 必要的时候提供 md 文档内的 code snippet 示例或者 mermaid 流程图。
- 必要的时候提供独立的示例代码 folder。
- 代码里的关键点，需要在代码里增加注释，用同样的行文风格进行说明。
- md 文件的排版尽量简洁，除了标题，就用 bullet points，不要随意加粗或者斜体。
- 下面每个主要知识点都作为一个独立 md 文件，并在这个 `root.md` 中引用。

# 学习大纲

- 大纲是按“从 C++/native 背景切入 Java 后端”这条线来排的，建议顺序阅读。
- 每个知识点尽量结合你已有的 C++/图形/native 背景做对比，帮助快速建立直觉。
- 建议直接装 JDK 17 或 21 学习。大部分单文件示例 JDK 11+ 即可 `java 文件名.java` 运行；教程里也会提到 `record`、新版 `switch`、虚拟线程等较新的语法/能力，用 JDK 17+ 更顺。
- 大部分知识点在文末配了可运行的独立示例（在 `examples/` 下，每个示例一个 folder），文档里有运行命令。

1. [运行原理](运行原理.md)：Java 是怎么编译和运行的，JVM、字节码、JIT，和 C++ 的“直接编译成机器码”有什么不同。✅
2. [基础语法](基础语法.md)：基本类型、变量、控制流、和 C++ 的语法差异点。✅
3. [面向对象](面向对象.md)：class、继承、接口、抽象类、封装、多态，以及和 C++ OOP 的区别。✅
4. [包与项目结构](包与项目结构.md)：package、import、classpath，以及标准后端项目目录怎么组织。
5. [泛型](泛型.md)：Java 泛型怎么用，和 C++ 模板的本质区别（类型擦除）。
6. [集合框架](集合框架.md)：List、Map、Set 等常用容器，对应 C++ STL 的心智模型。
7. [异常处理](异常处理.md)：异常体系、checked / unchecked、try-with-resources。
8. [IO与文件](IO与文件.md)：Path、Files、字节流/字符流、资源关闭，后端读写文件和网络资源的基础。
9. [内存与GC](内存与GC.md)：内存模型和垃圾回收，对比 C++ 手动管理 / RAII / 智能指针。
10. [并发](并发.md)：线程、锁、线程池、并发容器，后端高频考点。
11. [函数式与Stream](函数式与Stream.md)：Lambda、函数式接口、Stream 流式处理。
12. [后端常用类型](后端常用类型.md)：Optional、BigDecimal、java.time、enum、UUID 等日常后端常用标准库。
13. [构建与依赖](构建与依赖.md)：Maven / Gradle 怎么管理依赖和打包，对应 CMake 的心智模型。
14. [测试与日志](测试与日志.md)：JUnit、断言、测试目录、日志门面和日志级别。
15. [后端开发入口](后端开发入口.md)：把 Java 语言知识接到 HTTP API、JSON、数据库、配置和分层代码上。
