# 1、网络游戏的开端：Echo

- 协议
	- TCP：**TCP**（Transmission Control Protocol，传输控制协议）面向连接的可靠传输协议，像打电话一样，先建立连接再传输数据，确保数据一定到达但速度相对较慢
	- UDP：**UDP**（User Datagram Protocol，用户数据报协议）无连接的不可靠传输协议，像发快递一样，直接发送数据不管对方有没有收到，速度快但不保证数据完整
	- KCP：**KCP**（Kernel Control Protocol，一种快速可靠协议）基于UDP的可靠传输协议，保留了UDP的速度优势，同时通过重传和拥塞控制机制提高了可靠性，类似"加速版UDP"
	- HTTP：**HTTP**（HyperText Transfer Protocol，超文本传输协议）—— 用于Web浏览器和服务器之间传输网页数据的协议，特点是请求-响应模式，客户端发起请求，服务器返回响应，常用于REST API

- REST API：**REST API**（Representational State Transfer，表述性状态转移）一种基于 HTTP 协议设计的 Web API 风格，把网络上的数据都看作"资源"，通过 GET（查）、POST（增）、PUT/DELETE（改/删）这样的标准操作来访问和操作这些资源。REST 的核心思想很简单——就像一个图书馆系统，每本书就是一个"资源"，有一个唯一的编号（URL）
	- 想看书？用 GET 方法"借"
	- 想买书？用 POST 方法"购买"
	- 想换书？用 PUT 方法"更换"
	- 想扔书？用 DELETE 方法"扔掉"
	- **主要特点：**
		- 无状态：每次请求都是独立的，服务器不记住之前的请求
		- 统一接口：所有操作都通过 HTTP 标准方法进行
		- 层次化：客户端不需要知道服务器内部是怎么实现的

- **URL**（Uniform Resource Locator，统一资源定位符）—— 即"网址"，是互联网上每个资源（如网页、图片、视频）的唯一"门牌地址"，告诉浏览器去哪里找到并访问这个资源

- 网游的位置同步
	- 玩家1和玩家2都通过TCP、跟服务端连接
	- 玩家1移动 → 移动信息发给服务端 → 服务端发给玩家2 → 玩家2客户端收到玩家1的移动、完成玩家1的位置更新

- 分布式架构/集群：一台服务器不够，要多台服务器时，这些服务器之间，也用 TCP 来连接
